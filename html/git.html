Git

  • git status — показывает, какие файлы были изменены с момента последнего коммита, какие файлы готовы к коммиту, а какие файлы Git ещё не отслеживает.
  • git add — добавляет изменения из рабочего каталога в индекс (staging area) — промежуточную область перед созданием коммита. Можно добавлять файлы по одному, группами или все сразу.
  • git commit — фиксирует изменения, добавленные в индекс, в локальной истории репозитория. Каждый коммит — это снимок состояния проекта в определённый момент времени, снабжённый уникальным хешем и сообщением, описывающим суть изменений.
  • git branch — отображает перечень всех локальных веток проекта.
  • git checkout <ветка> — осуществляет переход на указанную ветку.
  • git checkout -b <ветка> — комбинированная команда: создаёт новую ветку и сразу же переключается на неё.
  • git merge <ветка> — объединяет выбранную ветку с текущей, добавляя её изменения.
  • git rebase — переписывает историю, перемещая коммиты одной ветки на конец другой, создавая линейную историю.
  • git pull — получает изменения из удалённого репозитория и автоматически сливает их с локальной веткой.
  • git push — передаёт сделанные локально изменения в связанный удалённый репозиторий для их сохранения и совместной работы.
  • git push -u origin <ветка> — отправка новой ветки и установка связи с origin (удалённым репозиторием).
